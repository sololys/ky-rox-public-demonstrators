# Claim Levels

This repository uses bounded claim levels.

## OPEN

Public, reproducible, inspectable.

Examples:

- deterministic software microtests
- reproducible run logs
- manifest/hash verification
- public diagrams
- high-level explanatory notes

## HOLD

Conceptually important, but not publicly asserted as validated here.

Examples:

- generalized selection-law language
- cross-domain analogies
- hardware mappings
- FPGA/interlock designs
- physical validation claims before hardware tests

## KILL

Not claimed by this repository.

Examples:

- physical validation without physical test
- product certification without external audit
- production safety claims
- new physics claims based only on software simulation
- disclosure of protected implementation details
